Chapter 8

This week I enjoyed learning more about forms in JavaScript. Some of the things that jumped out to me in chapter 8 were:

  • form.submit() submits the form automatically but will not trigger the "submit" event. We can submit the form manually by employing a button or input element with a type attribute of submit or even an input element with a type attribute of image.
  • form.reset() method will result in all the form controls reset back to their initial values specified in the HTML - reset buttons generally = poor usability. They are too easy to click.
    • focus: event occurs when element is focused on
    • Blur: event occurs when the user moves the focus away from the element
    • change : occurs when the user moves focus away from the element AFTER CHANGING IT

OOP

Object Oriented Programming : involves separating the code into objects that have properties and methods. Keeps related pieces of code encapsulated in objects that maintain state throughout the life of the program. Objects can be reused or modified.
3 main concepts

  1. Encapsulation
    • innerworkings kept hidden inside of the object and only the essential functionalities are exposed to the end user
  2. Polymorphism
    • same process can be used for different objects.
    • various objects can share the same method - but also have the ability to overrride shared methods with a more specific implementation
  3. Inheritance
    • taking the features of one object adn the adding some new features
    • take an object that already exists and then inherit all its properties and methods - improve functionality by adding new methods and properties
